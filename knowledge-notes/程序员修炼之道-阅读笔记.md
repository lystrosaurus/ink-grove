# 程序员修炼之道-阅读笔记

核心主题：软件工程与开发最佳实践

## 关键知识点
1. **DRY 原则**：每块知识应有单一权威表述——重复是万恶之源
2. **正交性**：组件互相独立——修改一个不影响其他
3. **曳光弹**：先写端到端简单版本再完善——类似 MVP 开发
4. **破窗理论**：烂代码不及时修复会引来更多烂代码
5. **橡皮鸭调试**：向别人解释问题时往往自己就找到了答案

## 深度理解
《程序员修炼之道》是软件工程领域的方法论经典。Andrew Hunt 和 Dave Thomas 总结了程序员职业生涯中的实用原则和最佳实践。"DRY 原则"（Don't Repeat Yourself）是全书最核心的原则：每一块知识在系统中都应该有一个单一、明确、权威的表述。重复是万恶之源——同样的逻辑出现在两个地方，修改时就会忘记改其中一个。"正交性"：系统中的组件应该互相独立——修改一个组件不应该影响其他组件。正交性好的系统易于测试、易于维护、易于扩展。"曳光弹"：先写一个端到端的简单版本，再逐步完善。这类似于 MVP 开发——先让系统跑起来，再添加功能。"破窗理论"在编程中的应用：一段烂代码如果不及时修复，会引来更多的烂代码。因为后来的开发者会想"反正这里已经很烂了，我随便写写也行"。"橡皮鸭调试法"：向别人（或一只橡皮鸭）解释你的代码问题，往往在解释过程中就能找到 bug。这是因为解释的过程迫使你把思路理清楚。

## 实践启发
- 写代码时时刻提醒自己：这段逻辑有没有在其他地方出现过——如果有，提取成函数
- 发现烂代码立即修复——不要等"有时间再说"
- 遇到 bug 时试着向别人解释——往往在解释过程中就找到了答案
