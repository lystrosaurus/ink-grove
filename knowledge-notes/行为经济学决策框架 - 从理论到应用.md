## 核心框架

### 理论基础：双系统模型

```
┌─────────────────────────────────────────────────────────┐
│                    人类决策双系统                          │
├──────────────────────┬──────────────────────────────────┤
│      系统 1(快思考)    │         系统 2(慢思考)             │
├──────────────────────┼──────────────────────────────────┤
│ 快速、自动、无意识       │ 缓慢、刻意、有意识                   │
│ 启发法、情感驱动         │ 逻辑分析、理性计算                   │
│ 容量大、持续运行         │ 容量有限、懒惰                       │
│ 易受框架影响            │ 可以校验系统 1                        │
└──────────────────────┴──────────────────────────────────┘
```

**关键洞察**: 系统 1 主导日常决策，系统 2 通常懒惰不干预。

## 主要认知偏差及其应对

### 1. 锚定效应

**现象**: 过度依赖第一条信息 (锚点)

**实验证据**:
- 《经济学人》订阅实验：诱饵选项改变 84% 人的选择
- 随机数字锚定影响后续估值

**应对策略**:
- 主动寻找多个参考点
- 问：没有这个信息我会如何判断？
- 货比三家，打破单一锚点

---

### 2. 损失厌恶

**现象**: 损失痛苦 ≈ 2× 同等收益快乐

**表现形式**:
- 过早卖出盈利股票，过晚卖出亏损股票
- 现状偏见，不愿改变
- 禀赋效应：高估已拥有物品

**应对策略**:
- 写下决策理由，定期复盘
- 采用外部视角
- 设定预设规则 (如止损线)

---

### 3. 社会规范 vs 市场规范

**现象**: 金钱激励可能挤出内在动机

**实验证据**:
- 托儿所罚款实验：罚款后迟到增加
- 家庭付钱做家务：关系受损

**应对策略**:
- 区分社交场景和交易场景
- 不用金钱衡量关系
- 谨慎使用物质激励

---

### 4. 默认选项力量

**现象**: 人们倾向于接受预设选项

**应用案例**:
- 器官捐献：默认同意 vs 默认拒绝 (99.98% vs 12%)
- 养老金：Save More Tomorrow (49%→86% 参与率)

**应对策略**:
- 审视默认选项是否符合自己利益
- 主动选择而非被动接受
- 利用默认选项建立好习惯

## 决策改进工具箱

### 事前验尸 (Pre-mortem)

**方法**: 在做决定前，假设项目失败，写下可能原因

**效果**: 减少过度自信和规划谬误

**模板**:
```
项目：[项目名称]
假设现在是一年后，项目彻底失败
失败原因可能包括:
1. 
2. 
3. 
针对每个原因的预防措施:
1. 
2. 
3. 
```

---

### 外部视角

**方法**: 查类似案例的统计数据，而非只看本案信息

**效果**: 减少规划谬误和过度自信

**示例**:
- 估计任务时间时查阅历史数据
- 投资决策前研究行业基准
- 创业前了解成功率统计

---

### 决策日志

**方法**: 记录决策过程、理由、预期结果，事后复盘

**效果**: 减少后见之明偏差

**模板**:
```
日期：
决策内容：
考虑因素：
信息来源：
预期结果：
实际结果 (事后填写):
学习:
```

---

### 检查清单

**方法**: 为重要决策领域制定检查清单

**效果**: 减少遗漏和启发法偏差

**示例清单** (投资决策):
- [ ] 是否受锚定影响？
- [ ] 是否考虑了反面证据？
- [ ] 是否有过度自信？
- [ ] 是否查了基础概率？
- [ ] 是否有损失厌恶影响？

## 助推应用：设计更好的选择环境

### 个人助推系统

#### 财务助推
```
自动储蓄系统:
1. 工资自动转入储蓄账户 (默认选项)
2. 加薪时自动提高储蓄比例
3. 投资默认分散化指数基金
4. 每月消费报告反馈
```

#### 健康助推
```
健康环境设计:
1. 健康食品放在显眼位置
2. 安装健康 APP 追踪数据
3. 加入运动社群 (社会规范)
4. 预约年度体检提醒
```

#### 学习助推
```
学习系统:
1. 自动加入学习计划
2. 公开学习承诺 (社会压力)
3. 设置每日 reminder
4. 学习时长追踪反馈
```

## 影响力防御手册

### 六大原理识别与防御

| 原理 | 识别信号 | 防御问题 |
|------|---------|---------|
| 互惠 | 免费礼物/帮助 | "这是真诚还是策略？" |
| 承诺一致 | 小承诺开始 | "我愿意坚持这个承诺吗？" |
| 社会认同 | "90% 的人都..." | "真实数据是什么？" |
| 喜好 | 过度赞美/套近乎 | "区分人与请求" |
| 权威 | 头衔/制服/标志 | "是真专家吗？诚实吗？" |
| 稀缺 | 限时限量 | "我真的需要吗？" |

## 整合应用：理性决策流程

```
┌──────────────────────────────────────────────────────────┐
│                   理性决策流程                              │
├──────────────────────────────────────────────────────────┤
│ 1. 意识到决策点：这是需要系统 2 的决策吗？                     │
│ 2. 启动系统 2: 放慢速度、列清单、查数据                        │
│ 3. 识别偏差：是否受锚定/框架/损失厌恶影响？                   │
│ 4. 寻求反方：主动寻找反面证据和不同观点                       │
│ 5. 外部视角：查类似案例统计数据                             │
│ 6. 事前验尸：假设失败，找原因                               │
│ 7. 记录决策：写下理由和预期，方便事后复盘                     │
│ 8. 设置反馈：建立追踪和复盘机制                             │
└──────────────────────────────────────────────────────────┘
```

## 与已读知识的连接

### 与《原则》的整合
- 原则是触发系统 2 的工具
- 决策日志减少后见之明
- 可信度加权帮助判断专家

### 与《心流》的整合
- 心流可能是系统 1 训练有素后的表现
- 专家直觉是系统 2 技能内化为系统 1
- 技能习得：系统 2 刻意练习→系统 1 自动执行

### 与《深度工作》的整合
- 深度工作需要系统 2 的持续投入
- 系统 2 容量有限，需要保护
- 减少干扰就是保护系统 2 资源
