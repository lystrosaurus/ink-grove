# 用户体验要素

状态：已读

作者：Jesse James Garrett

一句话评价：Jesse James Garrett 的"交互设计领域的方法论经典"——"提出了'用户体验的五层模型'"——"'战略层'——'范围层'——'结构层'——'框架层'——'表现层'"——"是'产品设计和开发的'必备指南'"。"'从抽象到具体'"——"'每一层'都'建立在'前一层'之上'"。"'战略层'（用户需求和产品目标）是'最基础的'一层"——"'没有清晰的战略'"——"'后面的设计'都是'空中楼阁'"。

核心收获：
- **五层模型**: Garrett 提出了"用户体验的五层模型"——"从'抽象'到'具体'"：1）**战略层**（Strategy）——"'用户需求'和'产品目标'"——"你要'解决什么问题'"——"'为谁解决'"；2）**范围层**（Scope）——"'功能规格'和'内容需求'"——"你要'做哪些功能'"——"'不做哪些'"；3）**结构层**（Structure）——"'交互设计'和'信息架构'"——"用户'怎么'使用你的产品"——"'信息怎么组织'"；4）**框架层**（Skeleton）——"'界面设计'——'导航设计'——'信息设计'"——"页面'长什么样'"；5）**表现层**（Surface）——"'视觉设计'"——"颜色、字体、图片"。"每一层'都'建立在'前一层'之上"——"'不能跳过'战略层'直接做'视觉设计"
- **战略层的重要性**: Garrett 说"战略层是'最基础'的一层"——"'没有清晰的战略'"——"'后面的设计'都是'空中楼阁'"。"很多产品'失败的原因'"——"不是'做得不好'"——"是'做错了东西'"——"'解决了'一个'不存在的问题'"。"在'战略层'"——"你要'明确两个问题'"：1）**用户需求**——"'你的用户'需要'什么'"；2）**产品目标**——"'你的产品'要'达到什么目的'"。"只有'这两个问题'都'回答了'"——"你'才能'进入'下一层'"
- **范围层的取舍**: Garrett 说"范围层的核心是'取舍'"。"'你不可能'做'所有功能'"——"'你必须'选择'做哪些'"——"'不做哪些'"。"他提出了'功能优先级'的方法'"——"'根据'用户需求'和'产品目标'"——"'给每个功能'打分"——"'得分高的'先做"——"'得分低的'不做'或者'以后做'"。"范围层的'另一个重要内容'"是"'内容需求'"——"'你的产品'需要'什么内容'"——"'这些内容'从'哪里来'"
- **结构层的设计**: Garrett 说"结构层的核心是'交互设计和信息架构'"。"交互设计'关注的是'"——"'用户'怎么'与产品'互动'"——"'点击一个按钮'会'发生什么'"——"'填一个表单'要'几步'"。"信息架构'关注的是'"——"'信息'怎么'组织'"——"'分类'怎么'划分'"——"'搜索'怎么'设计'"。"结构层'决定了'用户的'使用体验'"——"'如果结构不好'"——"'用户就会'迷路'"
- **框架层的细节**: Garrett 说"框架层是'最接近'用户'看到的东西'的一层"。"它包含'三个部分'"：1）**界面设计**——"'按钮'——'输入框'——'下拉菜单'等'界面元素'的'位置'"；2）**导航设计**——"'用户'怎么'在不同页面之间'切换'"；3）**信息设计**——"'信息'怎么'呈现'给'用户'"——"'用表格'还是'用图表'"——"'用文字'还是'用图片'"。"框架层的'目标'是'让用户'一眼就知道'怎么用'"
- **表现层的美学**: Garrett 说"表现层是'最后一层'"——"'也是'最'容易被'关注'的一层'"。"但'表现层'不是'孤立存在'的'"——"'它'依赖于'前四层'"。"'如果'战略层'没想清楚'"——"'表现层'再美'也没用'"。"表现层的'目标'是'传达'产品的'品牌气质'"——"'让用户'在'视觉上'就'感受到'产品的'价值'"
- **Jesse James Garrett 的背景**: Jesse James Garrett——美国交互设计专家——"Adaptive Path 公司联合创始人"——"'用户体验要素'（The Elements of User Experience）是他'最著名的作品'"——"'2002 年'首次出版"——"'成为了'交互设计领域'最经典的'教材之一"。"五层模型'被'广泛应用于'产品设计'——'网站设计'——'APP 设计'等'各个领域'"——"'是'每一个'产品经理和设计师'都'应该'了解'的'方法论'"
