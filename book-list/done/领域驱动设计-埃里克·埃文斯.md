# 领域驱动设计

状态：已读

作者：埃里克·埃文斯

一句话评价：埃里克·埃文斯（Eric Evans）的"软件工程里程碑之作'"——"'将业务领域建模的方法论系统化'"。"'限界上下文、通用语言、聚合根已成为现代架构设计的基石'"。"'微服务架构拆分的理论依据'"——"'消除开发与业务沟通鸿沟'"

核心收获：
- **限界上下文**: 本书核心是"'限界上下文（Bounded Context）'"的概念'"。"'什么'是'限界'上下文'"："'业务'的'边界'"——"'一个'模型'适用'的'范围'"。"'例子'"："'电商'系统'"："'订单'上下文'"——"'库存'上下文'"——"'支付'上下文'"。"'每个'上下文'"："'有'自己'的'模型'"——"'有'自己'的'术语'"。"'为什么'需要'"："'大'系统'中'"——"'同一'概念'在'不同'地方'含义'不同'"。"'比如'"："'产品'"："'销售'上下文'"——"'可'销售'"。"'仓储'上下文'"——"'有'库存'"。"'如何'划分'"："'根据'业务'边界'"——"'根据'团队'结构'"。"'限界'上下文'是'微服务'拆分'的'理论'依据'"
- **通用语言**: 本书"强调了通用语言的重要性'"。"'什么'是'通用'语言'"："'开发人员'和'领域'专家'使用'同一套'术语'"。"'问题'"："'开发'说'的'话'"——"'业务'不'懂'"。"'业务'说'的'话'"——"'开发'不'懂'"。"'如何'建立'"：1）**协作**——"'开发'和'业务'一起'讨论'"；2）**文档**——"'用'业务'术语'写'文档'"；3）**代码**——"'代码'中'用'业务'术语'命名'"。"'例子'"："'不'用'User'"——"'用'Customer'"——"'业务'说'的'词'"。"'好处'"："'消除'沟通'鸿沟'"——"'代码'更'易懂'"
- **聚合根**: 本书"介绍了聚合根的概念'"。"'什么'是'聚合根'"："'一组'相关'对象'的'根'"——"'外部'只能'访问'聚合根'"。"'例子'"："'订单'是'聚合根'"——"'订单项'是'内部'对象'"。"'外部'"："'只能'通过'订单'访问'"——"'不'能'直接'访问'订单项'"。"'为什么'"："'保证'一致性'"——"'简化'关系'"。"'聚合'的'原则'"：1）**一致性边界**——"'聚合'内'强'一致性'"；2）**小**——"'聚合'要'小'"——"'不'能'太大'"；3）**引用**——"'聚合'间'用'ID 引用'"
- **实体与值对象**: 本书"区分了实体与值对象'"。"'实体'"："'有'唯一'标识'"——"'状态'会变'"。"'例子'"："'用户'"——"'ID 不变'"——"'信息'会变'"。"'值对象'"："'没有'唯一'标识'"——"'不可变'"。"'例子'"："'地址'"——"'金额'"——"'颜色'"。"'如何'区分'"："'需要'唯一'标识'吗'"——"'需要'就是'实体'"。"'设计'原则'"："'尽量'用'值对象'"——"'值对象'更'简单'"——"'更'安全'"
- **领域模型**: 本书"强调了领域模型的重要性'"。"'什么'是'领域'模型'"："'业务'的'抽象'"——"'用'代码'表达'业务'逻辑'"。"'问题'"："'传统'开发'"——"'业务'逻辑'在'服务'层'"——"'模型'只是'数据'容器'"。"'DDD 的'方法'"："'业务'逻辑'在'领域'模型'中'"——"'模型'有'行为'"。"'例子'"："'订单'模型'"："'有'计算'总价'的'方法'"——"'有'验证'的'方法'"。"'好处'"："'代码'更'接近'业务'"——"'更'易'维护'"
- **战略设计**: 本书"讨论了战略设计'"。"'什么'是'战略'设计'"："'大'规模'的'设计'"——"'多个'限界'上下文'的'关系'"。"'上下文'映射'"："'上下游'关系'"——"'合作关系'"——"'共享'内核'"。"'例子'"："'订单'系统'依赖'用户'系统'"——"'订单'是'下游'"——"'用户'是'上游'"。"'如何'处理'关系'"："'防腐层'"——"'转换'模型'"——"'保护'自己'的'模型'"
- **作者的背景**: 埃里克·埃文斯（Eric Evans）——美国软件工程师、架构师——"《领域驱动设计》的作者'"。"他'在'思科'等'公司'工作'"——"'积累'了'大型'系统'经验'"。"《领域驱动设计》'2003 年'出版'"——"'蓝色'封面'"——"'被称为'蓝皮书'"。"'这本书'的'影响'"："'成为'企业'级'开发'的'经典'"——"'微服务'的'理论'基础'"。"'后来'沃恩'弗农'写'了'《实现'领域'驱动'设计》'"——"'更'实用'"
