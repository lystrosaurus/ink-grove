# 程序员修炼之道

状态：已读

作者：Andrew Hunt, David Thomas

一句话评价：Andrew Hunt 和 David Thomas 的"软件工程领域的方法论经典"——"总结了'程序员职业生涯中的'实用原则和最佳实践'"——"是'每位开发者都应该阅读的'手册'"。"'DRY 原则'（Don't Repeat Yourself）"——"'每一块知识'在系统中'都应该有'一个'单一、明确、权威的'表述'"。"'破窗理论'在编程中的应用"——"'一段烂代码'如果不及时修复'"——"'会引来'更多的'烂代码'"。"'橡皮鸭调试法'"——"'向别人'（或一只橡皮鸭）'解释你的代码问题'"——"'往往'在解释过程中'就能找到 bug'"

核心收获：
- **DRY 原则**: 作者提出了"'DRY 原则'"（Don't Repeat Yourself）——"'每一块知识'在系统中'都应该有'一个'单一、明确、权威的'表述'"。"'重复'是'万恶之源'"——"'当你'有'两段'相似的'代码'"——"'你'就'埋下了'隐患'"——"'因为'当'你'需要'修改'时'"——"'你'会'忘记'修改'另一段'"。"DRY 不仅是'关于代码'"——"'也是'关于'文档'"——"'关于'流程'"——"'关于'测试'"——"'任何'知识'都'不应该'重复'"
- **正交性**: 作者提出了"'正交性'"的概念——"'正交'意味着'独立性'"——"'如果'两个'模块'是'正交的'"——"'修改'一个'模块'"——"'不会'影响'另一个'模块'"。"'正交性'的'好处'"：1）**降低风险**——"'一个'地方'出问题'"——"'不会'波及'其他地方'"；2）**提高可维护性**——"'你'可以'单独'测试'和'修改'每个'模块'"；3）**促进复用**——"'正交的'模块'更'容易'被'复用'"。"作者说'正交性'是'好设计'的'标志'"
- **破窗理论**: 作者"将'破窗理论'应用到'编程中'"——"'一段烂代码'如果不及时修复'"——"'会引来'更多的'烂代码'"。"'如果'一个'项目'中'有'很多'烂代码'"——"'新来的'程序员'会'认为'这个'项目'不'重要'"——"'他们'会'随意'写'烂代码'"——"'最终'整个'项目'就'毁了'"。"作者说'发现'烂代码'要'立刻'修复'"——"'不要'让'破窗'存在'"
- **橡皮鸭调试法**: 作者介绍了"'橡皮鸭调试法'"——"'向别人'（或一只橡皮鸭）'解释你的代码问题'"——"'往往'在解释过程中'就能找到 bug'"。"'当你'向别人'解释'时'"——"'你'必须'组织'你的'思路'"——"'你'必须'清晰地'描述'问题'"——"'在'这个'过程中'"——"'你'会'发现'自己'的'逻辑错误'"。"作者说'橡皮鸭调试法'的'关键是'要'大声说出来'"——"'说'比'想'更'有效'"
- **追踪与日志**: 作者强调"'追踪与日志'的重要性'"。"'当'程序'出问题'时'"——"'你'需要'知道'程序'执行了'哪些'步骤'"——"'变量'的'值'是'什么'"——"'这'就是'追踪'"。"'日志'是'追踪'的'工具'"——"'你'应该'在'关键'位置'记录'日志'"——"'这样'当'问题'出现'时'"——"'你'可以'通过'日志'找到'根源'"。"作者说'好的'日志'应该'包含'时间戳、上下文、错误信息'"
- **测试驱动开发**: 作者"提倡'测试驱动开发'（TDD）"——"'先'写'测试'"——"'再'写'代码'"——"'最后'重构'"。"TDD 的'好处'"：1）**确保代码正确**——"'测试'通过'才'能'证明'代码'正确'"；2）**提供安全感**——"'当'你'修改'代码'时'"——"'测试'能'告诉你'是否'破坏了'现有功能'"；3）**改善设计**——"'为了'便于'测试'"——"'你'会'写出'更'模块化'的'代码'"。"作者说'测试'不是'负担'"——"'是'投资'"
- **Andrew Hunt 和 David Thomas 的背景**: Andrew Hunt 和 David Thomas——"资深软件工程师"——"《程序员修炼之道》的作者"——"这本书 1999 年出版"——"已经'更新了'第 2 版"——"是'软件工程领域'最'经典的'书籍之一"。"他们'提出了'很多'广为人知的'概念'"——"'如'DRY'——'正交性'——'橡皮鸭调试法'"——"'这些'概念'已经'成为'了'程序员'的'常识'"
