# 简约至上

状态：已读

作者：Giles Colborne

一句话评价：Giles Colborne 的"交互设计领域的经典指南'"——"'提出通过'删除、组织、隐藏、转移'四步实现简约设计'"——"'是'让复杂产品变得简单易懂的'实用手册'"。"'简约'不是'简单去掉功能'"——"'是'通过'删除、组织、隐藏、转移'四步'让产品更好用'"。"'删除'是'最重要的'一步'"——"'去掉不必要的功能'"——"'保留核心价值'"。"'好的简约设计'让'用户感觉理所当然'"——"'不需要说明书就能上手'"

核心收获：
- **简约的四步法**: Colborne 提出了"'实现简约的'四步法'"：1）**删除**——"'去掉'不必要的'功能'"——"'保留'核心'价值'"；2）**组织**——"'把'功能'分门别类'"——"'让'用户'容易找到'"；3）**隐藏**——"'把'不常用的'功能'隐藏起来'"——"'让'界面'更简洁'"；4）**转移**——"'把'复杂'的'操作'转移'到'合适的'地方'"——"'如'从'手机'转移到'电脑'"。"这四步'不是'孤立的'"——"'是'相互配合'的'"
- **删除**: Colborne 说"'删除'是'最重要的'一步'"。"'很多产品'的'问题'"——"'不是'功能太少'"——"'是'功能太多'"——"'用户'被'淹没'在'功能中'"——"'找不到'自己'需要的'功能'"。"'删除'意味着'"——"'去掉'不必要的'功能'"——"'保留'核心'价值'"。"'如何'决定'哪些'功能'该'删除'"——"'问'用户'"——"'他们'真正'需要'什么'"——"'而不是'问'我们'能'做什么'"
- **组织**: Colborne 说"'组织'是'第二步'"——"'把功能分门别类'"。"'如果'功能'太多'"——"'不能'都'删除'"——"'那就'组织'起来'"——"'让'用户'容易找到'"。"'组织'的'原则'"：1）**分块**——"'把'相关'的'功能'放在'一起'"——"'如'所有'设置'放在'设置菜单'"；2）**优先级**——"'常用的'功能'放在'显眼的'位置'"——"'不常用的'放在'次要'位置'"；3）**一致性**——"'不同页面'的'组织方式'要'一致'"——"'让'用户'形成'习惯'"
- **隐藏**: Colborne 说"'隐藏'是'第三步'"——"'把不常用的功能隐藏起来'"。"'有些功能'"——"'不常用'"——"'但'需要'时'必须'有'"——"'这些'功能'应该'隐藏起来'"。"'隐藏'的'方法'"：1）**渐进展开**——"'默认'隐藏'"——"'用户需要时'展开'"；2）**上下文相关**——"'在'合适的'场景'下'显示'"——"'如'编辑模式下'显示'删除按钮'"；3）**自定义**——"'让用户'自己'选择'显示哪些'功能'"。"'隐藏'不是'删除'"——"'是'让'界面'更简洁'"
- **转移**: Colborne 说"'转移'是'第四步'"——"'把复杂的操作转移到合适的地方'"。"'有些操作'"——"'在'手机'上'很复杂'"——"'在'电脑'上'很简单'"——"'那就'转移'到'电脑'"。"'转移'的'场景'"：1）**从设备转移到设备**——"'如'从'手机'转移到'电脑'"；2）**从用户转移到系统**——"'如'自动保存'——'自动同步'"；3）**从当下转移到未来**——"'如'稍后阅读'——'收藏'"。"'转移'的'目的'"——"'让'用户'在'合适的'场景'做'合适的'事'"
- **简约的原则**: Colborne 总结了"'简约的'几个原则'"：1）**聚焦核心价值**——"'产品'的'核心'价值'是什么'"——"'一切'围绕'这个'核心'"；2）**理解用户场景**——"'用户'在'什么场景'下'使用'产品'"——"'设计'要'适应'场景'"；3）**测试迭代**——"'简约'不是'一次'完成的'"——"'要'不断测试'——'不断迭代'"；4）**保持一致性**——"'简约'的'设计'要'保持一致'"——"'让'用户'形成'习惯'"
- **Giles Colborne 的背景**: Giles Colborne——英国交互设计师、用户体验专家——"《简约至上》的作者"——"2010 年出版"——"是'交互设计领域'的'经典指南'"。"他'有'20 多年'的'设计经验'"——"'服务过'很多'知名企业'"——"'他'的'设计哲学'是'以用户为中心'"——"'让'复杂'的'产品'变得'简单易懂'"。"《简约至上》的'特点'"——"'不'是'理论书'"——"'是'实用手册'"——"'有'大量的'案例'和'方法'"——"'可以'直接'应用到'设计工作中'"
