# 程序员修炼之道

状态：已读

作者：Andrew-Hunt

一句话评价：软件工程领域的方法论经典，总结了程序员职业生涯中的实用原则和最佳实践，是每位开发者都应该阅读的手册。

核心收获：
- 理解了"DRY 原则"（Don't Repeat Yourself）：每一块知识在系统中都应该有一个单一、明确、权威的表述
- "破窗理论"在编程中的应用：一段烂代码如果不及时修复，会引来更多的烂代码
- "橡皮鸭调试法"：向别人（或一只橡皮鸭）解释你的代码问题，往往在解释过程中就能找到 bug
