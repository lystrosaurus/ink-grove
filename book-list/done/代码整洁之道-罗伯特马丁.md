# 代码整洁之道

状态：已读

作者：罗伯特-马丁

一句话评价：程序员的"写作规范"——好代码不是给机器跑的，是给人读的。写代码和写文章一样，清晰比炫技重要一万倍。

核心收获：
- 变量名、函数名要"自解释"——读到名字就知道它是什么、做什么
- 函数只做一件事，且只做这一件事。函数超过20行就要警惕
- 注释是代码的失败——好代码不需要注释来解释它在做什么
- 代码坏味道：过长函数、过长参数列表、重复代码、过度耦合
- 重构不是"有空再做"，是持续的过程。每写一段代码都要问"能不能更整洁"
