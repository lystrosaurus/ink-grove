# 别让我思考

状态：已读

作者：史蒂夫·克鲁格

一句话评价：可用性设计领域的经典之作，提出网页设计的第一定律——"别让我思考"（Don't Make Me Think）。好的设计应该让用户不需要思考就能完成操作，这不是"降低用户智商"，是"尊重用户的时间"。

核心收获：
- **第一定律：别让我思考**: 用户打开一个网页或 App 时，不应该问"这是什么""我该点哪里""这个按钮是干什么的"。好的设计是"一目了然"的——用户不需要阅读说明、不需要学习、不需要猜测。"别让我思考"不是"用户蠢"，是"用户忙"——他们不想在你这里花脑力，他们想快速完成任务
- **扫描而非阅读**: 用户在网页上的行为不是"阅读"，是"扫描"——他们快速浏览页面，寻找和自己目标相关的线索。所以设计要"支持扫描"——用清晰的标题、醒目的按钮、分明的区块引导用户的视线。不要指望用户会仔细阅读你的文案
- **三个问题**: 用户打开任何页面时，潜意识里只有三个问题：1）这是什么网站？2）这里有什么？3）我该从哪里开始？好的设计必须在 3 秒内回答这三个问题。如果用户 3 秒内找不到答案，他们就关闭页面
- **可用性测试的极简方法**: 克鲁格说可用性测试不需要很多人——5 个用户就能发现 85% 的问题。测试不需要实验室、不需要专业设备、不需要复杂的流程——找一个会议室，录屏，让普通用户试用你的产品，观察他们在哪里卡住。这个方法的核心是"尽早测试，经常测试"
- **去除不必要的信息**: 设计新手最喜欢做的事是"加东西"——加功能、加文案、加装饰。克鲁格说好设计的关键是"减东西"——去掉一切不必要的元素。每多一个按钮，用户就多一次犹豫；每多一段文字，用户就多一分不耐烦
- **导航的清晰性**: 导航是网站的"路标"。好的导航应该让用户随时知道"我在哪""我能去哪""我怎么回去"。克鲁格建议：导航要固定位置（用户习惯在顶部或左侧找导航）、用用户语言（不要用"我们的解决方案"，用"产品"）、当前页面要高亮
- **"别让我思考"的普适性**: 这个原则不仅适用于网页设计，适用于一切需要与人交互的设计——App、软件、ATM 机、自动售票机、甚至实体产品的说明书。"别让我思考"的本质是"用户中心设计"——设计的目标不是"展示设计者的聪明"，是"让用户轻松完成任务"
- **这本书的影响力**: 《别让我思考》2000 年出版后成为了 UX 领域的"圣经"——几乎所有设计师都读过。它的影响力不在于"提出了新理论"，在于"用最简单的方式说清了最基本的道理"。很多设计师追求"炫酷"，但克鲁格提醒他们：炫酷不是目标，好用才是
