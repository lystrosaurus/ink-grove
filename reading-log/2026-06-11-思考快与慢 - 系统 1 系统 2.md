## 核心框架：双系统理论

### 系统 1 (快思考)
| 特征 | 说明 |
|------|------|
| 速度 | 快速、自动 |
| 意识 | 无意识或少意识 |
| 努力 | 不费力 |
| 控制 | 无法控制 |
| 例子 | 2+2=?、识别面孔、理解简单句子 |

### 系统 2 (慢思考)
| 特征 | 说明 |
|------|------|
| 速度 | 缓慢、刻意 |
| 意识 | 有意识 |
| 努力 | 费力 |
| 控制 | 可以控制 |
| 例子 | 17×24=?、填写表格、复杂计算 |

### 关键洞察
> 系统 1 持续运行，系统 2 通常懒惰。系统 1 产生印象和直觉，系统 2 负责验证和决策。

## 主要认知偏差

### 锚定效应 (Anchoring Effect)
- 人们过度依赖接收到的第一条信息 (锚)
- 即使锚明显不合理，也会影响判断
- 应用：谈判、定价、估计

### 可得性启发 (Availability Heuristic)
- 根据容易想起的例子判断频率
- 媒体曝光度影响风险感知
- 导致对罕见事件的过度估计

### 代表性启发 (Representativeness)
- 根据相似度判断概率
- 忽略基础概率
- 导致刻板印象

### 损失厌恶 (Loss Aversion)
- 损失带来的痛苦>同等收益带来的快乐
- 比例约 2:1
- 导致风险规避和现状偏见

### 框架效应 (Framing Effect)
- 同一问题的不同表述影响决策
- "存活率 90%"vs"死亡率 10%"
- 系统 1 受表面特征影响

### 过度自信 (Overconfidence)
- 高估自己的知识和能力
- 后见之明偏差
- 专家预测往往不准确

### 规划谬误 (Planning Fallacy)
- 低估完成任务所需时间
- 基于最佳情境规划
- 忽略历史数据

## 我的感想

### 核心震撼
- 原来大脑有两套系统在运行
- 系统 2 比想象中更懒惰
- 很多"理性决策"其实是系统 1 的直觉

### 自我反思
- 我是否过度相信自己的直觉？
- 我是否意识到自己的认知偏差？
- 我是否在重要决策时启动系统 2？

## 行动启发

### 决策改进
1. [ ] 重要决策前问自己：这是系统 1 还是系统 2？
2. [ ] 刻意启动系统 2：慢下来、列清单、求证据
3. [ ] 寻求外部观点，减少过度自信
4. [ ] 使用检查清单，减少锚定效应

### 偏差防范
1. [ ] 估计任务时间时参考历史数据
2. [ ] 考虑反面证据，减少确认偏误
3. [ ] 框架转换：从不同角度看问题
4. [ ] 事前验尸：假设失败，找原因

### 日常生活应用
1. [ ] 购物时警惕锚定效应
2. [ ] 投资时认识损失厌恶
3. [ ] 评估风险时查证真实概率
4. [ ] 做计划时考虑最坏情况
