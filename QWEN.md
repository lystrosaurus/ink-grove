# ink-grove — 项目上下文

## 项目概述

**ink-grove**（墨林）是一个个人知识管理仓库，用于系统化地管理阅读全生命周期：从选书、阅读记录到知识沉淀。项目名取"墨蕴成林，书卷为森"之意，强调日积月累、持续积累的长期主义。

- **类型**：非代码项目，纯 Markdown 知识库
- **所有者**：Opabinia
- **许可证**：MIT
- **创建日期**：2026-05-15

---

## 目录结构

```
ink-grove/
├── book-list/          # 书籍清单，按阅读状态分类
│   ├── to-read/        #   待读 — 想读还没开始的书
│   ├── reading/        #   在读 — 正在翻阅中的书
│   └── done/           #   已读 — 读完的书，可附简短评价
├── reading-log/        # 阅读记录，日常读书打卡与进度随笔
├── knowledge-notes/    # 知识笔记，摘录、感悟、干货知识点沉淀
├── archive/            # 归档专区，旧书单、往期笔记归档收纳
└── templates/          # 模板目录，统一笔记格式
```

## 命名规范

| 目录 | 文件命名格式 | 示例 |
|------|-------------|------|
| `book-list/*` | `书名-作者.md` | `置身事内-兰小欢.md` |
| `reading-log/` | `YYYY-MM-DD-书名-简述.md` | `2026-05-13-置身事内-第三章笔记.md` |
| `knowledge-notes/` | `书名-主题.md` | `置身事内-地方政府财政逻辑.md` |
| `archive/` | 沿用原目录命名 | — |

### 文件名字符规则

- 统一使用纯连字符 `-` 作为单词分隔符
- **禁止**使用以下字符：空格、` - `（带空格的连字符）、`·`（中点）、`_`（下划线）
- 书名/作者名内部已有的中文标点（如 `、` `·`）在文件名中一律替换为 `-`，但在正文和元数据中保持原样
- 多位作者用 `-` 连接（如 `贾雷德-戴蒙德`，不用 `贾雷德·戴蒙德`）
- 作者名应与仓库中已有文件保持一致（同一作者不可出现多种写法）

**正误对比**：

```
✅ 正确：三体2黑暗森林-刘慈欣.md
❌ 错误：三体 2 黑暗森林 - 刘慈欣.md    （含空格和带空格连字符）
❌ 错误：三体2黑暗森林 - 刘慈欣.md       （带空格连字符）

✅ 正确：人类简史-尤瓦尔-赫拉利.md        （中点替换为连字符）
❌ 错误：人类简史-尤瓦尔·赫拉利.md        （含中点 `·`）

✅ 正确：枪炮-病菌与钢铁-贾雷德-戴蒙德.md  （顿号和中点均替换）
❌ 错误：枪炮、病菌与钢铁-贾雷德·戴蒙德.md （含顿号和中点）
```

## 模板系统

`templates/` 目录包含三套标准模板，新建笔记时复制对应模板到目标目录即可：

| 模板文件 | 用途 | 使用场景 |
|---------|------|---------|
| `book-entry.md` | 书单条目 | 添加新书到 `book-list/to-read/` 时使用 |
| `reading-log.md` | 阅读记录 | 每次阅读后在 `reading-log/` 创建记录 |
| `knowledge-note.md` | 知识笔记 | 读完一本书或一个章节后，在 `knowledge-notes/` 沉淀核心要点 |

### 模板字段说明

**book-entry.md** — 书单条目包含：
- 基本信息（书名、作者、出版社、页数、ISBN、分类、领域细分）
- 来源信息（来源渠道、链接、添加日期、推荐人、优先级 P0/P1/P2）
- 阅读状态（待读→在读→已读，含日期追踪）
- "为什么想读"（加入书单的动机）
- "一句话评价"（读完后填写）
- 相关链接（关联知识笔记、阅读记录、其他书籍）

**reading-log.md** — 阅读记录包含：
- 日期与页码范围
- 今日摘录（块引用格式）
- 感想随笔
- 疑问与待查

**knowledge-note.md** — 知识笔记包含：
- 来源书籍与关联主题
- 核心要点（至少 3 个）
- 深入理解
- 实践启发（应用方式 + 行动计划）
- 延伸阅读

## 工作流指南

### 1. 添加新书

1. 复制 `templates/book-entry.md` → `book-list/to-read/书名-作者.md`
2. 填写书籍基本信息和"为什么想读"
3. 开始阅读时，将文件移动到 `book-list/reading/`
4. 读完后移动到 `book-list/done/`，补写"一句话评价"

### 2. 记录阅读

1. 复制 `templates/reading-log.md` → `reading-log/YYYY-MM-DD-书名-简述.md`
2. 随手记录摘录、感想、疑问，不必追求完整

### 3. 沉淀知识

1. 复制 `templates/knowledge-note.md` → `knowledge-notes/书名-主题.md`
2. 整理核心要点、深入理解、实践启发
3. 关联跨书籍的知识点，形成知识网络

### 4. 归档整理

- 已读完且笔记已沉淀的书单条目 → 移入 `archive/`
- 过期的阅读计划、废弃草稿 → 移入 `archive/`

## 阅读节奏

- **日常节奏**：每天 1 小时阅读，每本书 4-6 天完成
- **产出标准**：每本书 4-6 篇阅读记录 + 1 篇知识笔记
- **书单扩展**：读完一本书后，从延伸推荐、豆瓣、知乎等渠道补充新书

---

## 编辑规范

- **语言**：所有笔记正文使用中文；专有名词、技术术语保留英文原样
- **格式**：标准 Markdown（CommonMark 兼容），不使用 HTML 标签
- **标题层级**：`#` 文章主标题，`##` 主要章节，`###` 子章节
- **列表**：无序用 `-`，有序用 `1. 2. 3.`
- **引用**：摘录内容使用块引用 `>`
- **空行**：标题前后、段落之间保留一个空行
- **不添加 front matter**：本仓库不使用 YAML front matter

---

## 文件命名验证

定期检查命名一致性，保持仓库整洁：

- **触发时机**：每次批量添加书籍后、每季度例行整理时
- **检查范围**：`book-list/`、`reading-log/`、`knowledge-notes/` 三个目录
- **检查规则**：
  1. 文件名是否包含空格、` - `、`·`、`_` 等禁止字符
  2. 同一作者在不同文件中的写法是否一致
  3. `reading-log/` 和 `knowledge-notes/` 中引用的书名是否与 `book-list/` 中的文件名匹配
- **修正方式**：使用 `git mv` 重命名文件，确保 Git 历史可追溯

## 交互约定

当用户在本项目中进行以下操作时，请遵循对应的规范：

| 用户意图 | Agent 行为 |
|---------|-----------|
| "添加一本书" | 引导用户填写 `book-entry.md` 的关键字段（书名、作者、分类、想读理由）；**创建文件前检查目标目录是否已有同名或近似文件**，避免重复；生成符合命名规范的文件到 `book-list/to-read/` |
| "记录今天的阅读" | 复制 `reading-log.md` 模板，按 `YYYY-MM-DD-书名-简述.md` 生成文件名；**确认书名与 `book-list/` 中的文件名一致**，引导用户填写摘录与感想 |
| "整理读书笔记" | 复制 `knowledge-note.md` 模板，生成到 `knowledge-notes/`；**确认书名与已有书籍文件名一致**，引导结构化整理核心要点与实践启发 |
| "这本书读完了" | 将 `book-list/reading/` 中的对应文件移动到 `book-list/done/`，更新状态为"已读"，提示补写评价；询问是否需要创建知识笔记 |
| "归档旧内容" | 确认目标内容的知识笔记已沉淀，再执行移动操作到 `archive/` |
| "列出我的书单" | 分别列出 `to-read/`、`reading/`、`done/` 中的文件，按状态汇总 |
| "检查命名规范" | 扫描 `book-list/`、`reading-log/`、`knowledge-notes/` 所有目录，报告文件名中包含空格、` - `、`·` 等不规范字符的文件，并给出修正建议 |
