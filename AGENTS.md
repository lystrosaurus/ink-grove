## 项目概述

**ink-grove**（墨林）是一个纯 Markdown 格式的个人知识管理仓库，用于系统化地管理阅读全生命周期：从选书、阅读记录到知识沉淀。

- **类型**：非代码项目，纯 Markdown 知识库
- **所有者**：Opabinia
- **许可证**：MIT
- **自然语言**：中文（所有笔记内容使用中文，技术术语保留英文原样）

> 注意：本项目不含任何编程语言代码、构建脚本、包管理器配置或运行时依赖。不存在 `pyproject.toml`、`package.json`、`Cargo.toml`、`Makefile` 等文件。

## 目录结构

```
ink-grove/
├── book-list/          # 书籍清单，按阅读状态分类
│   ├── to-read/        #   待读 — 想读还没开始的书
│   ├── reading/        #   在读 — 正在翻阅中的书
│   └── done/           #   已读 — 读完的书，可附简短评价
├── reading-log/        # 阅读记录，日常读书打卡与进度随笔
├── knowledge-notes/    # 知识笔记，摘录、感悟、干货知识点沉淀
├── archive/            # 归档专区，旧书单、往期笔记归档收纳
└── templates/          # 模板目录，统一笔记格式
```

各子目录下均有一个 `README.md` 说明该目录的用途和规范。

## 命名规范

| 目录 | 文件命名格式 | 示例 |
|------|-------------|------|
| `book-list/*` | `书名-作者.md` | `置身事内-兰小欢.md` |
| `reading-log/` | `YYYY-MM-DD-书名-简述.md` | `2026-05-13-置身事内-第三章笔记.md` |
| `knowledge-notes/` | `书名-主题.md` | `置身事内-地方政府财政逻辑.md` |
| `archive/` | 沿用原目录命名 | — |

**禁止**在文件名中使用空格，统一使用连字符 `-` 分隔。

---

## 模板系统

`templates/` 目录包含三套标准模板。创建新笔记时，**必须**复制对应模板到目标目录并按规范重命名。

| 模板文件 | 用途 | 使用场景 |
|---------|------|---------|
| `book-entry.md` | 书单条目 | 添加新书到 `book-list/to-read/` 时使用 |
| `reading-log.md` | 阅读记录 | 每次阅读后在 `reading-log/` 创建记录 |
| `knowledge-note.md` | 知识笔记 | 读完一本书或一个章节后，在 `knowledge-notes/` 沉淀核心要点 |

## 核心工作流

### 1. 添加新书

1. 复制 `templates/book-entry.md` → `book-list/to-read/书名-作者.md`
2. 填写书籍基本信息（书名、作者、出版社、页数、分类）和"为什么想读"
3. 状态填写"待读"
4. 开始阅读时，将文件移动到 `book-list/reading/`，状态改为"在读"
5. 读完后移动到 `book-list/done/`，状态改为"已读"，补写"一句话评价"

### 2. 记录阅读

1. 复制 `templates/reading-log.md` → `reading-log/YYYY-MM-DD-书名-简述.md`
2. 填写日期、页码范围
3. 随手记录摘录、感想、疑问，不必追求完整

### 3. 沉淀知识

1. 复制 `templates/knowledge-note.md` → `knowledge-notes/书名-主题.md`
2. 整理核心要点、深入理解、实践启发
3. 关联跨书籍的知识点，形成知识网络
4. 填写"延伸阅读"建立链接

### 4. 归档整理

- 已读完且笔记已沉淀的书单条目 → 移入 `archive/`
- 过期的阅读计划、废弃草稿 → 移入 `archive/`
- **归档前必须确认**笔记已沉淀到 `knowledge-notes/`，避免知识丢失

## 代码风格与编辑规范

- **语言**：所有笔记正文使用中文；专有名词、技术术语保留英文原样。
- **格式**：标准 Markdown（CommonMark 兼容），不使用 HTML 标签。
- **标题层级**：一级标题 `#` 用于文章主标题，二级标题 `##` 用于主要章节，三级标题 `###` 用于子章节。
- **列表**：无序列表使用 `-`，有序列表使用 `1. 2. 3.`。
- **引用**：摘录内容使用块引用 `>`。
- **空行**：标题前后保留一个空行，段落之间保留一个空行。
- **不添加 front matter**：本仓库不使用 YAML front matter。

## 构建与测试

本项目无构建过程、无测试套件、无持续集成。

- **验证方式**：确保新增文件符合命名规范、使用正确的模板、Markdown 语法正确即可。
- **预览命令**：可使用任何 Markdown 预览工具本地查看，但无需提交任何构建产物。

## 安全与隐私

- 本项目为公开仓库（MIT 许可证），**不要在笔记中记录密码、密钥、个人敏感信息**。
- 书籍摘录请控制在合理使用范围内，避免大段复制受版权保护的内容。

## 常见操作指引

当用户在本项目中提出以下需求时，Agent 应按如下规范执行：

| 用户意图 | Agent 行为 |
|---------|-----------|
| "添加一本书" | 引导用户填写 `book-entry.md` 的关键字段（书名、作者、分类、想读理由），生成符合命名规范的文件到 `book-list/to-read/` |
| "记录今天的阅读" | 复制 `reading-log.md` 模板，按 `YYYY-MM-DD-书名-简述.md` 生成文件名，引导用户填写摘录与感想 |
| "整理读书笔记" | 复制 `knowledge-note.md` 模板，生成到 `knowledge-notes/`，引导结构化整理核心要点与实践启发 |
| "这本书读完了" | 将 `book-list/reading/` 中的对应文件移动到 `book-list/done/`，更新状态为"已读"，提示补写评价；询问是否需要创建知识笔记 |
| "归档旧内容" | 确认目标内容的知识笔记已沉淀，再执行移动操作到 `archive/` |
| "列出我的书单" | 分别列出 `to-read/`、`reading/`、`done/` 中的文件，按状态汇总 |
