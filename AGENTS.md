## 项目概述

**ink-grove**（墨林）是一个纯 Markdown 格式的个人知识管理仓库，用于系统化地管理阅读全生命周期：从选书、阅读记录到知识沉淀。

- **类型**：非代码项目，纯 Markdown 知识库
- **所有者**：Opabinia
- **许可证**：MIT
- **自然语言**：中文（所有笔记内容使用中文，技术术语保留英文原样）

> 注意：本项目不含任何编程语言代码、构建脚本、包管理器配置或运行时依赖。不存在 `pyproject.toml`、`package.json`、`Cargo.toml`、`Makefile` 等文件。

## 目录结构

```
ink-grove/
├── book-list/          # 书籍清单，按阅读状态分类
│   ├── to-read/        #   待读 — 想读还没开始的书
│   ├── reading/        #   在读 — 正在翻阅中的书
│   └── done/           #   已读 — 读完的书，可附简短评价
├── reading-log/        # 阅读记录，日常读书打卡与进度随笔
├── knowledge-notes/    # 知识笔记，摘录、感悟、干货知识点沉淀
├── archive/            # 归档专区，旧书单、往期笔记归档收纳
└── templates/          # 模板目录，统一笔记格式
```

各子目录下均有一个 `README.md` 说明该目录的用途和规范。

## 命名规范

| 目录 | 文件命名格式 | 示例 |
|------|-------------|------|
| `book-list/*` | `书名-作者.md` | `置身事内-兰小欢.md` |
| `reading-log/` | `YYYY-MM-DD-书名-简述.md` | `2026-05-13-置身事内-第三章笔记.md` |
| `knowledge-notes/` | `书名-主题.md` | `置身事内-地方政府财政逻辑.md` |
| `archive/` | 沿用原目录命名 | — |

### 字符规则

文件名中**只允许**使用中文字符、英文字母、数字和连字符 `-`。具体约束：

- **禁止空格**：文件名中不得出现任何空格
- **禁止 ` - `**：不得使用带空格的连字符 ` - `，只用纯连字符 `-`
- **禁止 `·`（中点）**：作者名中的间隔号用 `-` 替代
- **禁止标点**：顿号 `、`、逗号 `,`、冒号 `:`、引号 `"'` 等标点一律替换为 `-`
- **数字直接拼接**：书名中的数字紧接文字，无需加分隔符
- **多作者用 `-` 分隔**：多个作者之间统一用 `-` 连接

### 正确与错误示例对比

| ❌ 错误 | ✅ 正确 | 问题 |
|--------|---------|------|
| `三体 - 刘慈欣.md` | `三体-刘慈欣.md` | 带空格的连字符 |
| `三体2 黑暗森林-刘慈欣.md` | `三体2黑暗森林-刘慈欣.md` | 书名中含空格 |
| `人类简史-尤瓦尔·赫拉利.md` | `人类简史-尤瓦尔-赫拉利.md` | 含中点 `·` |
| `枪炮、病菌与钢铁-贾雷德·戴蒙德.md` | `枪炮-病菌与钢铁-贾雷德-戴蒙德.md` | 含顿号和中点 |
| `未来简史:从智神到智人.md` | `未来简史-从智神到智人.md` | 含冒号 |
| `活着（第2版）.md` | `活着-第2版.md` | 含括号 |

## 模板系统

`templates/` 目录包含三套标准模板。创建新笔记时，**必须**复制对应模板到目标目录并按规范重命名。

| 模板文件 | 用途 | 使用场景 |
|---------|------|---------|
| `book-entry.md` | 书单条目 | 添加新书到 `book-list/to-read/` 时使用 |
| `reading-log.md` | 阅读记录 | 每次阅读后在 `reading-log/` 创建记录 |
| `knowledge-note.md` | 知识笔记 | 读完一本书或一个章节后，在 `knowledge-notes/` 沉淀核心要点 |

## 核心工作流

### 1. 添加新书

1. **创建前检查**：扫描 `book-list/to-read/`、`book-list/reading/`、`book-list/done/` 是否已存在同书名文件（注意不同作者名写法，如 `赫拉利` vs `尤瓦尔-赫拉利`）
2. 如已存在，**不要创建新文件**，直接更新现有文件内容
3. 复制 `templates/book-entry.md` → `book-list/to-read/书名-作者.md`
4. 填写书籍基本信息（书名、作者、出版社、页数、分类）和"为什么想读"
5. 状态填写"待读"
6. 开始阅读时，将文件移动到 `book-list/reading/`，状态改为"在读"
7. 读完后移动到 `book-list/done/`，状态改为"已读"，补写"一句话评价"

### 2. 记录阅读

1. **创建前检查**：扫描 `reading-log/` 是否已有同一天同一本书的记录
2. 如已存在，**不要创建新文件**，直接更新现有记录
3. 复制 `templates/reading-log.md` → `reading-log/YYYY-MM-DD-书名-简述.md`
4. 填写日期、页码范围
5. 随手记录摘录、感想、疑问，不必追求完整

### 3. 沉淀知识

1. **创建前检查**：扫描 `knowledge-notes/` 是否已有同书名的主题笔记
2. 如已存在，**不要创建新文件**，直接在现有笔记中补充内容
3. 复制 `templates/knowledge-note.md` → `knowledge-notes/书名-主题.md`
4. 整理核心要点、深入理解、实践启发
5. 关联跨书籍的知识点，形成知识网络
6. 填写"延伸阅读"建立链接

### 4. 归档整理

- **归档前检查**：确认 `knowledge-notes/` 中已有该书的知识笔记，再执行归档
- 已读完且笔记已沉淀的书单条目 → 移入 `archive/`
- 过期的阅读计划、废弃草稿 → 移入 `archive/`
- **归档前必须确认**笔记已沉淀到 `knowledge-notes/`，避免知识丢失

## 代码风格与编辑规范

- **语言**：所有笔记正文使用中文；专有名词、技术术语保留英文原样。
- **格式**：标准 Markdown（CommonMark 兼容），不使用 HTML 标签。
- **标题层级**：一级标题 `#` 用于文章主标题，二级标题 `##` 用于主要章节，三级标题 `###` 用于子章节。
- **列表**：无序列表使用 `-`，有序列表使用 `1. 2. 3.`。
- **引用**：摘录内容使用块引用 `>`。
- **空行**：标题前后保留一个空行，段落之间保留一个空行。
- **不添加 front matter**：本仓库不使用 YAML front matter。

## 构建与测试

本项目无构建过程、无测试套件、无持续集成。

- **验证方式**：确保新增文件符合命名规范、使用正确的模板、Markdown 语法正确即可。
- **预览命令**：可使用任何 Markdown 预览工具本地查看，但无需提交任何构建产物。

### 命名验证

可定期使用以下方法批量检查文件命名是否合规：

**检查文件名是否包含空格**：文件名中不应出现空格，如发现需重命名。

**检查是否使用 ` - `（带空格连字符）而非 `-`（纯连字符）**：如有需替换为空格两侧的连字符对。

**检查是否存在重复/近似文件**：同一本书可能存在多个文件（不同作者名写法、不同分隔符），需合并为一个并归档其余。

## 常见操作指引

当用户在本项目中提出以下需求时，Agent 应按如下规范执行：

| 用户意图 | Agent 行为 |
|---------|-----------|
| "添加一本书" | 引导用户填写 `book-entry.md` 的关键字段（书名、作者、分类、想读理由），生成符合命名规范的文件到 `book-list/to-read/` |
| "记录今天的阅读" | 复制 `reading-log.md` 模板，按 `YYYY-MM-DD-书名-简述.md` 生成文件名，引导用户填写摘录与感想 |
| "整理读书笔记" | 复制 `knowledge-note.md` 模板，生成到 `knowledge-notes/`，引导结构化整理核心要点与实践启发 |
| "这本书读完了" | 将 `book-list/reading/` 中的对应文件移动到 `book-list/done/`，更新状态为"已读"，提示补写评价；询问是否需要创建知识笔记 |
| "归档旧内容" | 确认目标内容的知识笔记已沉淀，再执行移动操作到 `archive/` |
| "列出我的书单" | 分别列出 `to-read/`、`reading/`、`done/` 中的文件，按状态汇总 |

## 文件命名检查清单

每次创建或重命名文件时，Agent 必须逐项确认以下内容：

- ✅ 文件名无空格
- ✅ 分隔符为纯连字符 `-`，无 ` - `、`·`、`、`、`,`、`:` 等字符
- ✅ 目标目录无同名或近似文件（同一书名不同作者写法视为近似）
- ✅ 使用了正确模板（`book-entry.md` / `reading-log.md` / `knowledge-note.md`）
- ✅ 作者名与已有文件保持一致（如已有 `尤瓦尔-赫拉利`，新文件也必须用此写法）

任何一项未通过，先修正再创建。
